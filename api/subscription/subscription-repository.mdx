## Overview

The `SubscriptionRepository` interface provides a comprehensive API for managing subscription operations, including purchasing, restoring purchases, checking entitlements, and managing daily rolls for free users.

**Package:** `com.dadomatch.shared.feature.subscription.domain.repository`

## Methods

### getSubscriptionStatus

<ParamField path="getSubscriptionStatus" type="() -> Flow<SubscriptionStatus>">
  Get current subscription status as a Flow for reactive updates. This allows observers to receive real-time updates when the subscription status changes.
</ParamField>

**Returns:** `Flow<SubscriptionStatus>` - A Flow that emits subscription status updates

**Example:**
```kotlin
subscriptionRepository.getSubscriptionStatus()
    .collect { status ->
        // Handle subscription status updates
    }
```

**Returns:** `Result<SubscriptionStatus>` - Result with updated subscription status after purchase

### decrementDailyRolls

</ParamField>

### resetDailyRolls

<ParamField path="resetDailyRolls" type="suspend () -> Unit">
  Reset daily roll count. This should be called at midnight to refresh the daily roll allowance for free users.
</ParamField>

**Example:**
```kotlin
subscriptionRepository.resetDailyRolls()
```

### logIn

<ParamField path="logIn" type="suspend (userId: String) -> Result<SubscriptionStatus>">
  Link current user with RevenueCat ID. This associates the user's subscription data with their user account.
</ParamField>

**Parameters:**
- `userId` (String) - The user identifier to link with RevenueCat

**Returns:** `Result<SubscriptionStatus>` - Result with subscription status after login

**Example:**
```kotlin
val result = subscriptionRepository.logIn(firebaseUserId)
result.onSuccess { status ->
    // User logged in and linked with RevenueCat
}
```

---

### logOut

<ParamField path="logOut" type="suspend () -> Result<SubscriptionStatus>">
  Unlink user from RevenueCat ID. This disassociates the user's subscription data from their account.
</ParamField>

**Returns:** `Result<SubscriptionStatus>` - Result with subscription status after logout

**Example:**
```kotlin
val result = subscriptionRepository.logOut()
result.onSuccess { status ->
    // User logged out
}
```
