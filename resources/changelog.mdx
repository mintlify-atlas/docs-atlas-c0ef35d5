## Release Process

Releases are fully automated via GitHub Actions:

### Creating a Release

```bash
# Ensure main branch is up to date
git checkout main && git pull

# Create and push version tag
git tag v1.1.0
git push origin v1.1.0
```

### Automated Steps

The CI/CD workflow automatically:

1. 🏗️ Builds the Release XCFramework
2. 📦 Creates zip archive with framework
3. 🔐 Calculates SHA256 checksum for SPM
4. 🎉 Creates GitHub Release with artifacts
5. 📝 Updates `Package.swift` with download URL and checksum
6. 🔄 Updates `VERSION` file
7. 💾 Commits changes back to main branch

<Warning>
  Only tags pushed from the `main` branch will trigger releases. Tags from other branches are rejected.
</Warning>

## Deprecation Policy

<Accordion title="Understanding deprecation notices">

### Deprecation Timeline

1. **Announcement**: Feature marked as `@Deprecated` with replacement suggestion
2. **Grace Period**: Minimum 2 minor versions before removal
3. **Warning**: Additional warnings added in next major version
4. **Removal**: Deprecated feature removed in next breaking version

### Example

```kotlin
// Version 1.2.0 - Deprecation announced
@Deprecated(
    message = "Use newMethod() instead",
    replaceWith = ReplaceWith("newMethod()"),
    level = DeprecationLevel.WARNING
)
fun oldMethod() { }

// Version 1.3.0 - Still available with warning
// Version 1.4.0 - Still available with warning

// Version 2.0.0 - Removed
```

</Accordion>
